0 Core philosophy foundation ›

The ring is the message

A ping carries zero content. Its meaning is entirely borrowed from the pre-established relationship between sender and receiver.

Zero social debt

A ping is a transaction that closes itself. Got it — debt paid. Decline — debt respectfully refused. No lingering obligation.

Peripheral awareness

Information lives in the environment. You perceive the state — you don't check it. Color and rhythm over banners and text.

Consent is mutual

No relationship can be imposed unilaterally. Both parties agree to every meaning, timing parameter, and direction of authority.

The record is the referee

Both sides hold an identical immutable copy of every session. Neither party can rewrite history or dispute a timestamp.

Protocol over escape hatch

No early fallback to calls or messaging. Trust the protocol, train the users. The system enforces its own discipline.

2 Signal lifecycle — action logic ›

Ping 1 — normal

Single short haptic burst. 3-second full-screen takeover with expanding ripple rings. Collapses to ambient mini Orb after timer expires. Neutral gray Orb color. No DND override.

Ping 2 — noticed

Longer haptic pattern. Full-screen takeover again. Amber Orb. Receiver sees this is a follow-up, not first contact. Gap between pings logged in session timeline.

Ping 3 — urgent

Double pulse haptic. Full-screen. Red Orb. DP shakes subtly on receiver screen — urgency cue without text. DND override fires if mutually enabled. Face animation communicates "third time" before any reading.

Ping 4 — critical

Triple pulse. Full-screen. Crimson Orb. White jolt flash before crimson if DND override active. Session seals after this ping regardless of reply. Sender locked. Cooldown begins.

Session close conditions

Got it reply · Decline reply · Signal expiry · Ping 4 reached with no reply (critical seal) · Sender manually cancels before delivery. Each writes to history in its respective color.

New session after critical

Cooldown window enforced by system. When it lifts, sender can open a new session. Neutral color. No carry-over of previous escalation. Old session stays in history permanently — never deletable.

4 Relationship setup flow core ›

Step 1 — Who

Search existing users by name or @username. Or send a setup link to someone not yet on the app. Link pre-fills sender's proposed settings so receiver doesn't start from blank.

Step 2 — Signal type fork

Action or warmth. Everything downstream changes. Action path = 5 steps. Warmth path = 2 steps. Both can coexist between the same two people as separate relationships.

Step 3A — Roles and meanings (action)

Relationship type preset or custom. Superior role and executor role defined. Directional meaning set for each side. Executor's return ping is always a question, never a command — enforced by label in UI.

Step 3W — Shared meaning (warmth)

One meaning, both directions. No role labels. Palette color selection — purple, pink, teal, blue. Amber and red unavailable — those colors are owned by action signals.

Step 4A — Session behaviour (action)

Signal expiry · Reply window · Cooldown after ping 4 · DND override threshold (mutual opt-in only) · Max pings per day. All configurable per relationship, not global.

Step 4W — Soft limits (warmth)

Daily soft limit (default 3). Fade without reply toggle. Soft history toggle (count only). No expiry. No cooldown. No escalation settings — these concepts don't exist for warmth.

Step 5 — Confirm, send, negotiate

Full summary shown to sender before sending. Receiver sees all proposed settings with inline edit pills on any field they disagree with. Negotiate sends a counter-proposal. Accept locks the relationship live. Both sides must agree the same version — no unilateral activation.

5 Incoming ping UI interface ›

Full-screen takeover — 3 seconds

Dark full screen. Radial glow background matching session color. Large Orb with expanding ripple rings as the "ringing" visual. DP centered and breathing. Sender name and predefined message above. Timer bar draining at bottom. Got it and decline below.

Collapse to ambient — after 3 seconds

Full-screen shrinks to mini Orb and joins the live stack. Persistent amber banner at top of screen with contact name and one-tap reply access. Mini Orb continues breathing in session color. Nothing is lost — ping is still active and waiting.

Re-ping takeover difference

Same 3-second full-screen but ping count shown as color — amber for ping 2, red for ping 3, crimson for ping 4. DP shakes subtly on ping 3+. No digit shown — color alone carries the count.

Warmth ping difference

Soft purple/pink/teal glow instead of amber. Decline replaced with "Later" — no rejection language. "Felt" replaces "Got it" — warmer vocabulary. No timer pressure feeling — bar drains more gently.

In-pocket gesture response

Single back-tap = got it. Double back-tap = decline. Uses native iOS back-tap accessibility API and Android squeeze gesture. Haptic breath every 2 minutes while session is active — passive awareness without demanding attention.

6 Sender status screen interface ›

Live tab

Summary strip — got it / waiting / queued / critical counts. Active sessions list with mini Orbs, escalation dot clusters, and outcome labels. Acknowledged today section below active. Tap any item to open session detail.

Session detail screen

Contact name and role. Signal intent displayed. Full ping timeline with timestamps to the second. Gap between pings shown in italic. Expiry progress bar live. Re-ping button active or locked depending on state. Cooldown countdown when locked.

History tab

Chronological list grouped by month. Color bar on left of each entry. Tap to expand full session ping timeline with gap notes. Immutable — no deletion. Both sender and receiver hold identical copies.

Received tab

Pings sent to the user as receiver. Their reply record. Timestamps immutable. This is the tab that protects the receiver from false accusations. Reply timestamps shown to the second.

7 Relationship history view trust ›

Timeline tab

Full session history since inception. Grouped by month. Color bar per session. Tap to expand. Each expanded view shows full ping-by-ping breakdown with gap between pings in italic. Seconds-level timestamps on all events.

Stats tab

Got it / declined / expired / critical totals. Response rate bar chart by month. Relationship health visible at a glance without any editorial commentary — just the numbers.

Meanings tab

Current active meanings with effective dates. Full meaning change history with previous wording and change date. Solves handshake drift — neither party can dispute what the ping meant at any point in time.

Ping-back count

Private count of sessions that went unanswered and were marked as ignored. Visible only to sender against that specific relationship. Not punitive — informational. Tells the sender: this person has a pattern.

Identity anchor — submerged DP

Four layers: dynamic glow / frosted glass / identity ring / quiet DP. DP is static — only opacity and saturation shift with state. Grayscale at idle, full color when active, pulse ring at critical. Two-stage brain processing: peripheral = urgency color, focused = identity face. Under one second total.

Zero-banner phone UI

No slide-down notification banners. Always-on display shows blurred Orb glow in session color. Edge glow on lock screen visible from across a room. Dynamic Island carries live session dot persistently on iOS. Screen edges pulse softly while session active.

Physical Orb — Phase 3

Frosted glass desktop sphere. Bluetooth LE. Single tap = got it. Cover = decline. Flip = mute session. No buttons, no screen, no speakers. Observer pattern — receives STATE_UPDATE from app logic layer. Color and breathing pattern only interface.

Enterprise Orb network — Phase 4

Multiple Orbs on one network. Manager sees 12 Orbs on a surface — each one a crew member's live signal state. Ambient operations visibility. No screens, no dashboards, no checking. The room itself is the interface.

9 Contextual badges interface ›

Allowed badge types

Location pin · ETA (5 / 10 / 30 min) · Floor or room number · Urgency flag · Time slot. Maximum 3 badge types per relationship — configured at setup, not free-form.

Never allowed

Video · document · image · voice note · free text · link · emoji. Any attachment that can say something new — rather than sharpen the existing meaning — is messaging. That line holds permanently.

Interaction

Long-press on the ping button before sending opens the badge tray. Tap once to attach. One badge per ping maximum. Badge appears alongside the predefined message on receiver's full-screen takeover.
