Ping session model and dialled history view showing escalation states and session resets

session logic

Session opens

Sender sends first ping. Fresh state, neutral color. Session ID assigned silently in background.

Session escalates

Each unanswered re-ping deepens the color. Ping count tracked within session only — invisible to receiver until ping 4.

Session closes at ping 4

Critical red reached. Session is sealed and written to history. Sender is locked. No more pings within this session — ever.

New session = clean slate

Next ping from sender opens a brand new session. Normal color. No carry-over of previous escalation. The past is history, not baggage.

A session can also close cleanly — got it, declined, or expired. Those write to history too, in their respective colors. Only the critical session carries red in the log.
